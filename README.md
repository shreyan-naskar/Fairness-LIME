## Design Details

### Dataset and Feature Engineering

The **Bank Marketing** dataset (45,211 instances, 16 attributes) is used to predict whether a client subscribes to a term deposit.

**Interpretable features** (6 binary, used for explanation):

| Feature       | Condition                                     |
| ------------- | --------------------------------------------- |
| `age_bin`     | age ≥ 39 (also the **sensitive attribute** z) |
| `balance_bin` | balance > 448                                 |
| `housing_bin` | has housing loan                              |
| `edu_bin`     | education is primary or secondary             |
| `marital_bin` | marital status is married                     |
| `loan_bin`    | has personal loan                             |

**Non-interpretable features** (37-dimensional): one-hot encodings of job, contact type, month, and poutcome; plus numeric day, duration, campaign, pdays, previous, and a default flag. These are z-score normalised using training statistics only.

The full feature vector fed to the model is the 43-dimensional concatenation (interpretable + non-interpretable), with a bias prepended (44 dimensions total).

**Train/test split**: 80/20, random seed 42 (numpy RandomState).

---

### Task X — Model Training and Instance Selection

Two logistic regression classifiers are trained from scratch (no scikit-learn):

#### Model-Accurate (Covariance Projection, `c = 0.8`)

Trained with projected gradient descent. After every gradient step, the weight vector is projected onto the constraint set `|Cov(z, X·w)| ≤ c` via a closed-form projection along the covariance gradient direction. This keeps the model correlated with the sensitive attribute within a loose bound, prioritising accuracy.

Hyperparameters: 3000 iterations, learning rate 0.08, L2 regularisation 1e-4.

#### Model-Fair (Lagrangian Dual, `γ = 0`)

Minimises the absolute covariance with `age_bin` directly. Per-sample Lagrange multipliers enforce that each instance's loss stays within `(1 + γ)` times its unconstrained loss. The dual variables are updated by subgradient ascent and clipped to `[0, 200]`.

Hyperparameters: 1500 iterations, weight learning rate 0.05, dual learning rate 0.1, L2 regularisation 1e-4.

#### Instance Selection

Five test instances are selected:

- **2 disagreement instances**: Model-Accurate and Model-Fair predict different labels.
- **3 agreement instances**: Both models predict the same label.

---

### Task Y — Neighbourhood Generation

For each of the 5 selected instances, **10 perturbed samples** are generated by independently and uniformly randomising each of the 6 interpretable binary features (random seed 123). The 37 non-interpretable features are held **constant** at the original instance's values. Both models then predict class probabilities for each reconstructed sample, producing `(5 × 10)` probability arrays for each model.

---

### Task Z — LIME Local Explanations

A LIME-style weighted linear regression is fit per instance per model, entirely in the interpretable feature space.

**Step 1 — Hamming distance**: Count the number of binary features that differ between each perturbed sample and the original instance.

**Step 2 — Exponential kernel weights**:

```
w(z') = exp(- D(x', z)^2 / σ^2),   σ = sqrt(6) ≈ 2.449
```

Closer neighbours receive higher weight.

**Step 3 — Weighted ridge regression**: Solve the weighted least-squares problem (with ridge `ε = 1e-6`) to fit a linear model mapping interpretable features to black-box probabilities.

**Step 4 — Local prediction**: Evaluate the surrogate at the original instance to get a local probability estimate comparable to the model's actual output.

---

### Task Z — Submodular Feature Selection

Global feature importance is extracted from LIME by stacking the 5 per-instance absolute coefficient vectors into a `(5 × 6)` matrix `A`.

The **coverage** of a feature subset `S` is:

```
F(S) = Σ_i sqrt( Σ_{j ∈ S} A[i,j]^2 )
```

This function is monotone submodular. A **greedy algorithm** iteratively adds the feature that gives the largest marginal coverage gain until 3 features are selected, achieving a `(1 - 1/e)` approximation guarantee.
